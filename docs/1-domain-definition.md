# 도메인 정의서 — TodoListApp

**버전:** 1.0  
**작성일:** 2026-05-12  
**작성자:** Business Analyst

---

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 앱 이름 | TodoListApp |
| 목적 | 인증된 사용자 단위로 개인 할일을 카테고리별로 관리하는 웹 애플리케이션 |
| 핵심 가치 | 개인화된 할일 관리, 카테고리 기반 분류, 데이터 영속성 보장 |

### 문제 정의

1. 개인별 할일 데이터를 서버 데이터베이스에 안전하게 저장하고 관리하는 수단이 부족하다.
2. 인증된 사용자 단위로 데이터를 격리하여 프라이버시를 보장하는 할일 앱이 없다.
3. 사용자 정의 카테고리를 통해 할일을 유연하게 분류할 수 있는 앱이 없다.

---

## 2. 핵심 도메인 개념 (Ubiquitous Language)

| 용어 (한국어) | 용어 (영문) | 정의 |
|---------------|-------------|------|
| 사용자 | User | 회원가입 후 인증을 통해 앱을 사용하는 주체. 자신의 데이터에만 접근 가능하다. |
| 인증 | Authentication | 이메일과 비밀번호 기반의 신원 확인 절차. 인증되지 않은 사용자는 할일에 접근할 수 없다. |
| 할일 | Todo | 사용자가 수행해야 할 작업 단위. 제목, 설명, 종료예정일, 카테고리, 완료 여부를 가진다. |
| 카테고리 | Category | 할일을 분류하는 레이블. 시스템 제공 기본 카테고리와 사용자 정의 카테고리로 구분된다. |
| 기본 카테고리 | Default Category | 시스템이 모든 사용자에게 공통 제공하는 카테고리 (예: 업무, 개인, 쇼핑). 수정·삭제 불가. |
| 사용자 정의 카테고리 | Custom Category | 특정 사용자가 직접 생성한 카테고리. 해당 사용자에게만 보이며 수정·삭제 가능. |
| 종료예정일 | Due Date | 할일을 완료하기로 계획한 날짜. 기간별 필터링의 기준이 된다. |
| 완료 여부 | Completion Status | 할일의 완료(Done) 또는 미완료(Pending) 상태. |
| 할일 목록 | Todo List | 사용자가 등록한 할일 전체 집합. 카테고리, 기간, 완료 여부로 필터링 가능. |

## 3. 핵심 도메인 모델

### 3.1 주요 엔티티 및 속성

#### User (사용자)
| 속성 | 타입 | 설명 |
|------|------|------|
| id | UUID | 식별자 (PK) |
| email | String | 로그인 이메일, 유일값 |
| password_hash | String | 암호화된 비밀번호 |
| name | String | 사용자 표시 이름 |
| created_at | Timestamp | 가입일시 |
| updated_at | Timestamp | 정보 수정일시 |

#### Category (카테고리)
| 속성 | 타입 | 설명 |
|------|------|------|
| id | UUID | 식별자 (PK) |
| name | String | 카테고리 이름 |
| is_default | Boolean | 기본 카테고리 여부 |
| user_id | UUID \| NULL | 사용자 정의 카테고리의 소유자 (기본 카테고리는 NULL) |
| created_at | Timestamp | 생성일시 |

#### Todo (할일)
| 속성 | 타입 | 설명 |
|------|------|------|
| id | UUID | 식별자 (PK) |
| user_id | UUID | 소유 사용자 (FK → User) |
| category_id | UUID | 분류 카테고리 (FK → Category) |
| title | String | 할일 제목 (필수) |
| description | String \| NULL | 상세 설명 (선택) |
| due_date | Date \| NULL | 종료예정일 (선택) |
| is_completed | Boolean | 완료 여부 (기본값: false) |
| created_at | Timestamp | 등록일시 |
| updated_at | Timestamp | 수정일시 |

### 3.2 엔티티 관계

```
User (1) ──────────── (N) Todo
User (1) ──────────── (N) Category [Custom]
Category (1) ────────── (N) Todo
```

- 한 사용자는 여러 개의 할일을 소유한다.
- 한 사용자는 여러 개의 사용자 정의 카테고리를 생성할 수 있다.
- 하나의 카테고리에는 여러 할일이 속할 수 있다.
- 할일은 반드시 하나의 카테고리에 속해야 한다.

## 4. 주요 비즈니스 규칙

### 인증 및 접근 제어
- BR-01: 미인증 사용자는 할일 및 카테고리 데이터에 접근할 수 없다.
- BR-02: 사용자는 자신이 소유한 할일과 카테고리에만 접근·수정·삭제할 수 있다.
- BR-03: 이메일은 전체 시스템에서 유일해야 하며 로그인 식별자로 사용된다.

### 카테고리 규칙
- BR-04: 기본 카테고리는 모든 사용자에게 공통 제공되며, 사용자가 수정하거나 삭제할 수 없다.
- BR-05: 사용자 정의 카테고리는 해당 사용자만 조회·수정·삭제할 수 있다.
- BR-06: 할일은 반드시 하나의 카테고리(기본 또는 사용자 정의)에 속해야 한다.

### 할일 규칙
- BR-07: 할일 제목은 필수 입력값이다.
- BR-08: 종료예정일은 선택값이며, 입력 시 과거 날짜도 허용한다(과거 기록 등록 지원).
- BR-09: 완료 처리된 할일은 삭제하지 않고 상태 변경으로 관리한다.

### 필터링 규칙
- BR-10: 할일 목록 조회 시 카테고리, 기간(종료예정일 범위), 완료 여부 필터를 단독 또는 복합 적용할 수 있다.
- BR-11: 기간 필터는 종료예정일 기준으로 적용하며, 종료예정일이 없는 할일은 기간 필터 적용 시 제외된다.

## 5. 핵심 유스케이스 (사용자 스토리)

### 인증

| ID | 사용자 스토리 | 인수 조건 |
|----|---------------|-----------|
| UC-01 | 새 사용자로서, 이메일과 비밀번호로 회원가입하여 서비스를 이용하고 싶다. | 이메일 중복 검증, 비밀번호 암호화 저장, 가입 후 즉시 로그인 가능 |
| UC-02 | 기존 사용자로서, 이메일과 비밀번호로 로그인하여 내 할일에 접근하고 싶다. | 인증 성공 시 토큰 발급, 실패 시 명확한 오류 메시지 제공 |
| UC-03 | 인증된 사용자로서, 내 이름 등 개인 정보를 수정하고 싶다. | 본인 정보만 수정 가능, 변경 후 즉시 반영 |

### 카테고리 관리

| ID | 사용자 스토리 | 인수 조건 |
|----|---------------|-----------|
| UC-04 | 인증된 사용자로서, 시스템 기본 카테고리 목록을 확인하고 싶다. | 모든 사용자에게 동일한 기본 카테고리 노출 |
| UC-05 | 인증된 사용자로서, 나만의 카테고리를 추가하여 할일을 원하는 방식으로 분류하고 싶다. | 카테고리 이름 필수, 생성 즉시 할일 등록 시 선택 가능 |
| UC-06 | 인증된 사용자로서, 내가 만든 카테고리를 수정하거나 삭제하고 싶다. | 본인 생성 카테고리만 수정·삭제 가능, 기본 카테고리는 변경 불가 |

### 할일 관리

| ID | 사용자 스토리 | 인수 조건 |
|----|---------------|-----------|
| UC-07 | 인증된 사용자로서, 제목·설명·종료예정일·카테고리를 포함한 할일을 등록하고 싶다. | 제목 필수, 나머지 선택, 카테고리 미선택 시 기본 카테고리 자동 지정 |
| UC-08 | 인증된 사용자로서, 내 할일 목록을 카테고리·기간·완료 여부로 필터링하여 조회하고 싶다. | 필터 미선택 시 전체 목록 반환, 복합 필터 지원 |
| UC-09 | 인증된 사용자로서, 등록한 할일의 내용을 수정하고 싶다. | 본인 할일만 수정 가능, 수정 후 updated_at 갱신 |
| UC-10 | 인증된 사용자로서, 할일을 완료 처리하거나 미완료로 되돌리고 싶다. | 완료 여부 토글, 데이터 삭제 없이 상태만 변경 |
| UC-11 | 인증된 사용자로서, 더 이상 필요 없는 할일을 삭제하고 싶다. | 본인 할일만 삭제 가능, 삭제 후 목록에서 즉시 제거 |

## 6. 도메인 경계 (Bounded Context)

```
┌─────────────────────────────────────────────────────────┐
│                    TodoListApp                          │
│                                                         │
│  ┌──────────────────┐     ┌───────────────────────┐    │
│  │  Identity &      │     │   Todo Management     │    │
│  │  Access          │────>│                       │    │
│  │                  │     │  - Todo CRUD          │    │
│  │  - 회원가입       │     │  - 카테고리 관리       │    │
│  │  - 로그인/로그아웃 │     │  - 필터링 조회         │    │
│  │  - 개인정보 수정  │     │  - 완료 상태 관리      │    │
│  └──────────────────┘     └───────────────────────┘    │
│          │                          │                   │
│          └──────────────────────────┘                   │
│               인증 컨텍스트가 할일 컨텍스트에             │
│               사용자 신원(user_id)을 제공                 │
└─────────────────────────────────────────────────────────┘
```

### Identity & Access Context (인증/접근 관리 컨텍스트)
- 책임: 사용자 등록, 인증, 개인정보 관리
- 핵심 엔티티: User
- 외부 인터페이스: 인증 토큰 발급 (JWT 등)

### Todo Management Context (할일 관리 컨텍스트)
- 책임: 할일 및 카테고리의 생성·조회·수정·삭제, 필터링
- 핵심 엔티티: Todo, Category
- 의존성: Identity & Access Context로부터 인증된 user_id를 수신하여 데이터 격리 보장

### 컨텍스트 간 통합 규칙
- 할일 관리 컨텍스트는 인증 컨텍스트의 user_id만을 참조하며, User 내부 상세 정보에는 직접 접근하지 않는다.
- 모든 할일·카테고리 작업은 유효한 인증 토큰이 있을 때만 허용된다.
