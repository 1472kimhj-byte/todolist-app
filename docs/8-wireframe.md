### 상호작용 및 화면 전환

**이메일/비밀번호 검증:**
- 빈 필드: 폼 제출 불가 (버튼 비활성화)
- 유효하지 않은 이메일 형식: 필드 아래 에러 메시지 "올바른 이메일을 입력해주세요."
- 비밀번호 8자 미만: 필드 아래 에러 메시지 "비밀번호는 8자 이상이어야 합니다."

**화면 전환:**
- 로그인 성공 → 할일 목록 페이지로 이동
- 로그인 실패 → 폼 상단에 에러 배너 "로그인 실패. 이메일과 비밀번호를 확인해주세요."
- 회원가입 링크 → 회원가입 페이지로 이동

**API 호출:**
- `POST /api/auth/login`
- 요청: `{ email, password }`
- 응답: `{ accessToken, refreshToken, user: { id, email, name } }`

### 상호작용 및 화면 전환

**필터 상호작용:**
- "전체" 탭 클릭: 모든 할일 표시
- "진행중" 탭 클릭: 미완료 할일만 표시
- "완료" 탭 클릭: 완료된 할일만 표시
- 카테고리 드롭다운: 특정 카테고리의 할일만 표시
- 기간 필터: 선택한 기간 내의 할일만 표시

**할일 아이템 상호작용:**
- 체크박스 클릭: 완료/미완료 토글
- [✎] 버튼 클릭: 할일 수정 모달 열기
- [✕] 버튼 클릭: 할일 삭제 (확인 팝업)

**사이드바 상호작용:**
- 카테고리 클릭: 해당 카테고리의 할일만 표시 (필터 적용)
- "+ 카테고리 추가" 클릭: 카테고리 추가 폼 모달 열기

**네비게이션:**
- "프로필" 메뉴 클릭: 프로필 편집 페이지로 이동
- "로그아웃" 버튼 클릭: 로그아웃 확인 팝업 → 로그인 페이지로 이동

**API 호출:**
- `GET /api/todos?status=all&category=all&dateRange=all` — 할일 목록 조회
- `PATCH /api/todos/:id` — 완료 상태 토글
- `DELETE /api/todos/:id` — 할일 삭제
- `GET /api/categories` — 카테고리 목록 조회

```
┌────────────────────────────────────────┐
│ 할일 추가                          [✕] │
├────────────────────────────────────────┤
│                                        │
│ 제목 *                                 │
│ [_________________________________]   │
│ 최대 100자                            │
│                                        │
│ 설명                                   │
│ [_________________________________     │
│  _________________________________]   │
│ (최대 500자)                          │
│                                        │
│ 마감일                                 │
│ [날짜 선택] YYYY-MM-DD                │
│                                        │
│ 카테고리                               │
│ [업무 ▼]                              │
│                                        │
│                  [ 저장 ]  [ 취소 ]    │
│                                        │

```

**설명:**
- 모달 제목: "할일 추가"
- 필드:
  - 제목 (필수): 텍스트 입력, 100자 제한
  - 설명 (선택): 텍스트 영역, 500자 제한
  - 마감일 (선택): 날짜 선택 입력
  - 카테고리 (선택): 드롭다운 (기본값: 선택 또는 "업무")
- 버튼: [저장] (활성), [취소]

**설명:**

### 상호작용 및 검증

**필드 검증:**
- 카테고리 이름: 필수, 1자 이상, 50자 이하
- 중복 검사: 기존 카테고리와 이름 중복 불가

**에러 메시지:**
- 이름 빈 필드: "카테고리 이름을 입력해주세요."
- 이름 중복: "이미 존재하는 카테고리 이름입니다."
- 기타 API 에러: 모달 상단 에러 배너

**버튼 상태:**
- 이름 입력 전: [저장] 비활성화
- 이름 입력 후: [저장] 활성화

**화면 전환:**
- [저장] 클릭 → 카테고리 목록 업데이트 → 모달 닫기
- [취소] 또는 [✕] 클릭 → 모달 닫기 (저장 안 함)
- [삭제] 클릭 → 확인 팝업 "정말 삭제하시겠습니까? (해당 카테고리의 할일은 '업무' 카테고리로 이동됩니다.)" → 카테고리 삭제 → 모달 닫기

**API 호출:**
- 신규 추가: `POST /api/categories` — `{ name }`
- 수정: `PATCH /api/categories/:id` — `{ name }`
- 삭제: `DELETE /api/categories/:id`

```
  └─→ [회원 탈퇴] → 확인 팝업 → 로그인 페이지
```

## 설계 고려사항

### 반응형 브레이크포인트

| 디바이스 | 너비 | 레이아웃 |
|---------|------|---------|
| 모바일 | 320px ~ 767px | 단일 열, 스택 레이아웃 |
| 태블릿 | 768px ~ 1023px | 사이드바 + 콘텐츠 (조정됨) |
| 데스크톱 | 1024px+ | 사이드바 + 콘텐츠 (고정) |

### 색상 및 스타일 지침

**버튼 상태:**
- **활성 (활성):** 파란색 배경, 흰색 텍스트, `[ 텍스트 ]`
- **비활성 (disabled):** 회색 배경, 회색 텍스트, `[ 텍스트 ]`
- **위험 액션 (삭제, 탈퇴):** 빨간색 배경, 흰색 텍스트, `[ 텍스트 ]`

**입력 필드:**
- 기본: `[_________________]`
- 포커스: `[═════════════════]` (테두리 강조)
- 에러: `[_________________]` 아래에 빨간색 에러 메시지
- 성공: `[_________________]` 아래에 초록색 성공 메시지

**카테고리 배지:**
- 기본 카테고리: 파란색 배경 `[업무]`, 초록색 배경 `[개인]`, 주황색 배경 `[쇼핑]`
- 사용자 카테고리: 자동 색상 할당

**지나간 마감일:**
- ASCII에서: `2026-05-14 [!]` (느낌표 표시)
- 실제 UI: 빨간색 텍스트

**완료된 할일:**
- 체크박스: `☑` (체크됨)
- 제목: ~~취소선~~ 적용
- 텍스트 색상: 회색

### 접근성 (Accessibility)

- 모든 버튼과 링크는 포커스 가능해야 함
- 폼 필드에는 명확한 라벨이 있어야 함 (시각적 + 텍스트)
- 에러 메시지는 시각적 표시(색상)와 텍스트로 전달
- 모달 열 때 배경 요소 비활성화 (a11y 고려)
- 토스트 메시지는 스크린 리더로 읽을 수 있어야 함

### 성능 고려사항

- 할일 목록은 가상 스크롤링으로 렌더링 (많은 항목이 있을 경우)
- 이미지/아이콘은 SVG 또는 CSS로 처리
- 모달 열 때 백그라운드 네트워크 요청 최소화
- TanStack Query의 캐싱 활용으로 불필요한 API 호출 최소화
